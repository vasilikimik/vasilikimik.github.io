CPU Reliability Testing Framework

An MSc thesis project: design and implementation of a multi-core CPU reliability testing framework that detects hardware defects through carefully crafted arithmetic workloads. Built in C with a modular Built-In Self-Test (BIST) methodology and per-core fault detection via parallel execution.

C BIST IEEE 754 Fault Models pthreads Multi-Core Linux CPU Affinity
overview

Project Description

The goal of this framework is to detect manufacturing and ageing defects inside a CPU's arithmetic units by running targeted test patterns — arithmetic workloads designed to sensitise specific hardware faults and propagate their effect to an observable output.

The framework focuses on the floating-point datapath (IEEE 754), targeting the FP Multiplier and FP Adder and their internal sub-units. Test patterns are derived from classic test-generation theory (controllability & observability, D-Algorithm, PODEM) and executed in parallel across cores, with results compared via checksums to isolate per-core faults.

Framework Architecture

01
Modular C structure — shared declarations (bist.h), IEEE 754 utilities and core check routines (helpers.c), and a menu-driven interface (main.c).
02
Test phases — separate phase files implementing distinct pattern strategies: orthogonal Cartesian testing, edge-case coverage walks, trapezoid accumulation, anti-masking sparse patterns, sticky & rounding bounds, full-width Cartesian tiling, and pure random (fixed seed).
03
Multi-core execution — each thread is pinned to a specific core via CPU affinity (cpu_set_t, pthread_setaffinity_np), allowing the same workload to run independently per core.
04
Fault detection — results are compared via XOR checksums across cores; a mismatch isolates a per-core hardware fault.
